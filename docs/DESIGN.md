# zWork Design System

## Color Strategy
**Restrained** - Tinted neutrals + one accent ≤10%

### Primary Colors
- **Primary**: Blue (#4285F4) - Used sparingly for key actions and branding
- **Surface**: Paper tones (light/dark variants) - Main background and card colors
- **Text**: Ink tones (primary, muted, faint) - Text hierarchy
- **Border**: Line tones - Subtle separators and dividers

### Color Roles
- **Surface**: Main backgrounds and cards
- **Primary**: Google OAuth buttons, active states
- **Success**: Success messages and confirmations
- **Error**: Error messages and validation states

## Typography
- **Display**: Large headings with proper hierarchy
- **Body**: Clear, readable text with appropriate line lengths
- **Labels**: Small, informative text for UI elements

### Font Hierarchy
- **Headings**: 32px, 24px, 18px - Bold weight
- **Body**: 16px - Regular weight
- **Labels**: 14px, 12px - Medium weight
- **Micro**: 11px - Light weight

## Layout Principles
- **Mobile-first**: Responsive design that works on all screen sizes
- **Clear hierarchy**: Visual distinction between content sections
- **Adequate spacing**: Consistent rhythm between elements
- **No unnecessary containers**: Let content breathe where appropriate

## Component Styles
- **Cards**: Subtle elevation with soft shadows
- **Buttons**: Clear interactive states with hover and active feedback
- **Forms**: Clear labels and proper input styling
- **Navigation**: Simple, predictable navigation patterns

## Theme
- **Light**: Light paper tones with subtle gray accents
- **Dark**: Dark paper tones with appropriate contrast
- **Physical Scene**: "Developer working in a well-lit office or focused home environment, needing to maintain concentration while accessing AI assistance"

## Motion
- **Micro-interactions**: Subtle hover states and transitions
- **Animations**: Purposeful motion to guide user attention
- **Timing**: 150-300ms duration with ease-out curves
- **No bouncing**: Natural, smooth motion only

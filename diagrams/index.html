📌 Key Points

Active code used in production
Skeleton code (not wired to chat endpoint)
  • The entire agentic loop is in providers.py::stream_chat()
  • Multi-turn tool loop (max 24 turns) happens inside stream_chat()
  • core/ directory exists but is not called by /api/chat/stream
  • Tools are defined in tools.py (same dir as providers.py)

Data Flow Sequence (Actual)

sequenceDiagram
    participant U as User
    participant R as React UI
    participant S as server.py
    participant P as providers.py
    participant T as tools.py
    participant L as LLM API

    U->>R: Send message
    R->>S: POST /api/chat/stream
    S->>P: stream_chat(messages, model)
    P->>P: lookup_model(), resolve_creds()

    loop Multi-turn tool loop (max 24)
        P->>L: POST /messages (with tools)
        L-->>P: SSE stream (text + tool_use)

        loop Each tool_use block
            P->>T: execute_tool(name, params)
            T->>P: activity events (UI updates)
            T->>P: tool_result event
            P->>R: SSE: activity + result
            R->>U: Show tool progress
        end

        P->>P: Append tool_result to messages
    end

    P->>S: yield "done" event
    S->>R: SSE close
    R->>U: Display final response
    P->>P: chatstore.save()

📌 Tool Facts

  • TOOL_SCHEMAS is defined in tools.py (not imported from elsewhere)
  • Providers imports: from .tools import TOOL_SCHEMAS, execute_tool
  • Tools are NOT in the core/ directory — they're in agent/
  • No permission gate — tools execute directly when LLM calls them

📌 What This Means

The core/ directory contains a more sophisticated architecture with: planning phases, permission gates, and step-by-step execution. However, none of this is wired to the actual chat endpoint. The production path goes straight from server.py to providers.stream_chat().
